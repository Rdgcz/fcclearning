What is CSS?

Cascading Style Sheets (CSS) is a markup language used to apply styles to HTML elements. CSS is used for colors, background images, layouts and more.

Basic Anatomy of a CSS Rule

A CSS rule is made up of two main parts: a selector and a declaration block. A selector is a pattern used in CSS to identify and target specific HTML elements for styling. A declaration block applies a set of styles for a given selector, or selectors.

meta name="viewport" Element

This meta element gives the browser instructions on how to control the page's dimensions and scaling on different devices, particularly on mobile phones and tablets.

Default Browser Styles

Each HTML element will have default browser styles applied to them. This usually includes items like default margins and paddings.

CSS Basics

Cascading Style Sheets (CSS) is a stylesheet language used to describe the presentation of HTML documents. CSS controls how elements are displayed on web pages, including colors, layout, fonts, and responsive behavior across different devices.

Basic Anatomy of a CSS Rule

selector {
  property: value;
  another-property: value;
}

A CSS rule consists of a selector and a declaration block containing property-value pairs.

Viewport Meta Element

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This meta tag controls how a page is displayed on mobile devices, ensuring proper scaling across different screen sizes.

Default Browser Styles

Every browser applies default styles to HTML elements (margins, paddings, font sizes). These vary between browsers, which is why CSS resets or normalizations are often used.

CSS Specificity

CSS specificity determines which styles are applied when multiple rules target the same element:

  • Inline CSS: Highest specificity (1,0,0,0)
  • ID Selectors: (0,1,0,0)
  • Class Selectors: (0,0,1,0)
  • Type Selectors: (0,0,0,1)
  • Universal Selector: (0,0,0,0)

CSS Combinators

/* Descendant Combinator */
div p { }

/* Child Combinator */
div > p { }

/* Next-sibling Combinator */
h1 + p { }

/* Subsequent-sibling Combinator */
h1 ~ p { }

CSS Inheritance

The process by which styles are passed down from parent elements to their children. Inheritance allows you to define styles at a higher level in the document tree and have them apply to multiple elements without explicitly specifying them for each element.

Applying CSS

Inline CSS

<p style="color: blue; font-size: 16px;">This is a paragraph.</p>

Styles applied directly to HTML elements using the style attribute. Most of the time you will not be using inline CSS due to separation of concerns.

Internal CSS

<head>
  <style>
    p {
      color: blue;
      font-size: 16px;
    }
  </style>
</head>

These styles are written within the <style> tags inside the head section of an HTML document. This can be useful for creating short code examples, but usually, you will not use internal CSS.

External CSS

<head>
  <link rel="stylesheet" href="styles.css">
</head>

These styles are written in a separate CSS file and linked to the HTML document using the link element in the head section. For most projects, you will use an external CSS file over internal or inline CSS.

Box Model & Layout

The CSS Box Model

In the CSS box model, every element is surrounded by a box. This box consists of four components:

  • Content Area: The innermost part containing actual content
  • Padding: Space between content and border
  • Border: Outer edge or outline of the element
  • Margin: Space outside border between elements
Content Box

This element demonstrates the box model with padding, border, and margin.

box-sizing Property

/* Default */
box-sizing: content-box;

/* Recommended for easier sizing */
box-sizing: border-box;

With content-box, width and height only include content. With border-box, width and height include content, padding, and border but not margin.

Margin Collapse

This behavior occurs when the vertical margins of adjacent elements overlap, resulting in a single margin equal to the larger of the two. This behavior applies only to vertical margins (top and bottom), not horizontal margins (left and right).

Display Types

Type Behavior Examples
Inline Flow within content, don't start on new lines, can't set width/height span, a, img, strong
Block Take full available width, start on new lines, can set width/height div, p, section, h1-h6
Inline-Block Flow like inline but can set width/height button, input, select

Width and Height Properties

.element {
  width: 300px;
  min-width: 200px;
  max-width: 500px;
  height: 150px;
  min-height: 100px;
  max-height: 300px;
}

Layout & Positioning

Position Property

Value Description
static Default position in normal document flow
relative Positioned relative to its normal position
absolute Positioned relative to nearest positioned ancestor
fixed Positioned relative to viewport (stays on scroll)
sticky Toggles between relative and fixed based on scroll position

Floats

img {
  float: left;
  margin-right: 15px;
}

Floats are used to remove an element from its normal flow on the page and position it either on the left or right side of its container. When this happens, text will wrap around that floated content.

z-index Property

Controls the vertical stacking order of positioned elements that overlap. Higher values appear in front of lower values.

CSS Transform Property

.element {
  transform: translate(50px, 100px); /* Move element */
  transform: scale(1.5); /* Scale element */
  transform: rotate(45deg); /* Rotate element */
}

Typography

Font Properties

body {
  font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
  font-size: 16px;
  font-weight: 400;
  line-height: 1.6;
  color: #333;
}

Typography Terminology

  • Baseline: Imaginary line on which most characters rest
  • Cap Height: Height of uppercase letters
  • X-height: Average height of lowercase letters
  • Ascenders: Parts of letters extending above x-height
  • Descenders: Parts of letters extending below baseline
  • Kerning: Space adjustment between specific character pairs
  • Tracking: Space adjustment between all characters
  • Leading: Vertical space between lines of text

Web Safe Fonts

  • Sans-serif: Arial, Verdana, Trebuchet MS (good for screens)
  • Serif: Times New Roman, Georgia (good for print)

Custom Fonts with @font-face

@font-face {
  font-family: 'MyFont';
  src: url('myfont.woff2') format('woff2'),
       url('myfont.woff') format('woff');
  font-weight: 400;
  font-style: normal;
}

Text Shadow

h1 {
  text-shadow: 2px 2px 4px rgba(0, 0, 0, 0.5);
}

The text-shadow property applies shadows to text. You need to specify the X and Y offset, which represent the horizontal and vertical distance of the shadow from the text, respectively.

Color Theory

Color Relationships

  • Primary Colors: Red, Green, Blue (fundamental hues)
  • Secondary Colors: Orange, Green, Purple (mixed from primaries)
  • Tertiary Colors: Yellow-green, Blue-green, etc. (primary + secondary)

Color Schemes

Scheme Description Effect
Analogous Adjacent colors on wheel Cohesive, soothing
Complementary Opposite colors High contrast, visual impact
Triadic Three equidistant colors Vibrant, balanced
Monochromatic Variations of single color Unity, harmony

CSS Color Notation

color: red;                      /* Named color */
color: #ff0000;                  /* Hexadecimal */
color: rgb(255, 0, 0);           /* RGB */
color: rgba(255, 0, 0, 0.5);     /* RGB with alpha */
color: hsl(0, 100%, 50%);        /* HSL */
color: hsla(0, 100%, 50%, 0.5);  /* HSL with alpha */

Gradients

/* Linear Gradient */
background: linear-gradient(135deg, red, blue);

/* Radial Gradient */
background: radial-gradient(circle, red, blue);

Responsive Design

Media Queries

/* Mobile-first approach */
.container {
  width: 100%;
}

/* Tablet */
@media (min-width: 641px) {
  .container {
    width: 80%;
    margin: 0 auto;
  }
}

/* Desktop */
@media (min-width: 1025px) {
  .container {
    width: 70%;
  }
}

Common Breakpoints

Device Type Breakpoint Range
Smartphones Up to 640px
Tablets 641px to 1024px
Desktops 1025px and larger

Responsive Images

img {
  max-width: 100%;
  height: auto;
}

Relative Units

.element {
  width: 50%;        /* Percentage of parent */
  font-size: 1.5rem; /* Relative to root element */
  height: 50vh;      /* 50% of viewport height */
  margin: 2em;       /* Relative to element's font size */
}

calc() Function

.element {
  width: calc(100% - 40px);
  height: calc(50vh + 2rem);
}

Flexbox

CSS Flexbox is a one-dimensional layout model for arranging items in rows or columns.

Flex Container Properties

.container {
  display: flex;
  flex-direction: row; /* or column */
  flex-wrap: nowrap; /* or wrap */
  justify-content: flex-start; /* alignment along main axis */
  align-items: stretch; /* alignment along cross axis */
}

CSS Grid

CSS Grid is a two-dimensional layout system for creating complex web page layouts.

Grid Container Properties

.container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr; /* three equal columns */
  grid-template-rows: 100px auto; /* two rows */
  gap: 20px; /* space between grid items */
}

Animations & Transitions

Transitions

.button {
  transition: all 0.3s ease-in-out;
}

.button:hover {
  transform: scale(1.05);
  background-color: #2ecc71;
}

@keyframes slidein {
  from {
    transform: translateX(-100%);
  }
  to {
    transform: translateX(0);
  }
}

.element {
  animation: slidein 0.5s ease-in-out;
}

Animation Properties

.element {

  animation-duration: 0.5s;
  animation-timing-function: ease-in-out;

Accessibility

Focus Styles

button:focus {
  outline: 2px solid #3498db;
  outline-offset: 2px;
}

Always provide visible focus indicators for keyboard navigation.

Reduced Motion

@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}

Respect user preferences for reduced motion to prevent accessibility issues.

Screen Reader Only Content

.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

Color Contrast

Ensure that the background and foreground colors have good contrast to make the text readable. The Web Content Accessibility Guidelines (WCAG) recommend a minimum contrast ratio of 4.5:1 for normal text and 3:1 for large text.

Best Practices

CSS Methodology

  • Use consistent naming conventions (BEM, SMACSS, etc.)
  • Organize CSS in logical sections
  • Comment your code for maintainability
  • Use CSS variables for theming and consistency

Performance

  • Minify CSS for production
  • Use efficient selectors (avoid overly specific ones)
  • Reduce use of expensive properties like box-shadow and border-radius when possible
  • Leverage CSS containment for performance optimization

CSS Variables (Custom Properties)

:root {
  --primary-color: #3498db;
  --spacing: 1rem;
  --border-radius: 8px;
}

.button {
  background-color: var(--primary-color);
  padding: var(--spacing);
  border-radius: var(--border-radius);
}

Design Best Practices

  • Use subtle, intentional effects, honor preferences, and offer user control
  • Avoid content that flashes more than three times per second
  • Ensure good visual hierarchy in design
  • Use progressive enhancement approach

Pro Tip: Always test your CSS in multiple browsers and devices to ensure consistent appearance and behavior.
